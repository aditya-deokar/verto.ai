# Glossary

> Definitions of terms, abbreviations, and concepts used throughout the Verto AI codebase and documentation.

---

## A

### ADR (Architecture Decision Record)
A short document that captures a significant architectural decision, the context in which it was made, and its consequences. Stored in `docs/adr/`.

### Agent
A specialized AI function within the LangGraph pipeline. Each agent handles one step (e.g., outline generation, content writing). Verto AI uses 8 agents. See [03-agentic-workflow.md](03-agentic-workflow.md).

### App Router
Next.js 13+ routing system based on the file system. Uses `app/` directory with `page.tsx`, `layout.tsx`, and route groups `(parenthesized)`.

---

## B

### Background Job
A long-running task executed outside the main request cycle. In Verto AI, Inngest handles background jobs for mobile design generation.

---

## C

### Channel (LangGraph)
A typed state field in a LangGraph `StateGraph`. Channels define how state updates are merged when agents return partial state. Verto AI uses "last write wins" for all channels.

### ContentItem
The recursive data structure representing a single element within a slide. Each `ContentItem` has a `type` (heading, paragraph, image, etc.), `content` (children or text), and optional styling properties. See [04-data-model.md](04-data-model.md#contentitem-recursive).

### ContentType
The union type of all possible `ContentItem.type` values. Includes 28+ types: `column`, `heading1`-`heading4`, `paragraph`, `image`, `table`, `bulletedList`, `codeBlock`, `calloutBox`, etc.

### CUID
Collision-resistant Unique Identifier. Used by Prisma for model IDs (except User, which uses UUID). Generated by `cuid()`.

---

## D

### Dashboard
The main protected page (`/dashboard`) showing the user's projects, recent activity, and quick actions.

### Database Persister
The 8th and final agent in the generation pipeline. Saves the compiled slide JSON to the PostgreSQL database.

---

## E

### Edge (LangGraph)
A connection between two nodes (agents) in a LangGraph graph. Edges can be unconditional (`addEdge`) or conditional (`addConditionalEdges`).

### EventSource
The browser Web API for receiving Server-Sent Events. Used by `useStreamingGeneration` hook to receive real-time generation progress.

---

## F

### Fallback Image Provider
An image provider that returns categorized placeholder images when the Unsplash API is unavailable or not configured. Categories include business, technology, people, nature, abstract, data, education.

---

## G

### Generation Run
A `PresentationGenerationRun` record tracking a single execution of the AI pipeline. Stores status, progress, current step, error, and step snapshots.

### Graph (LangGraph)
A `StateGraph` instance defining the agent pipeline. Nodes are agents, edges define execution order, and state flows through channels.

---

## H

### HMR (Hot Module Replacement)
Turbopack's ability to update the browser instantly when source files change, without a full page reload.

---

## I

### Inngest
An event-driven serverless function platform used for background job processing. Handles mobile design generation in Verto AI.

### Inngest Function
A background function triggered by an event. Supports step functions (discrete, retryable steps), automatic retries, and observability.

---

## J

### JSON Compiler
The 7th agent in the pipeline and the most complex (~54KB). Maps structured content + layout type into the recursive `ContentItem` tree format used by the editor.

---

## L

### LangGraph
A stateful, graph-based framework for orchestrating multi-agent AI workflows. Built on LangChain. Verto AI uses it for the 8-agent presentation generation pipeline.

### Layout Selector
The 3rd agent in the pipeline. Selects the best visual layout template for each slide based on its outline topic. Runs BEFORE content writing (layout-first architecture).

### Layout Template
A predefined slide structure defining component arrangement, image requirements, and content structure type. Examples: `twoColumn`, `comparisonLayout`, `bigNumberLayout`, `quoteLayout`.

### Lemon Squeezy
A Merchant of Record service handling payment processing, tax collection, and compliance for subscription billing.

---

## M

### MasterRecursiveComponent
The core rendering component that walks a `ContentItem` tree and renders the appropriate React component for each node type. Handles editor, presentation, export, and share rendering contexts.

### Merchant of Record (MoR)
A payment provider that takes legal responsibility for selling, including tax collection, compliance, and invoicing. Lemon Squeezy is Verto AI's MoR.

### Middleware
Next.js Edge Middleware (`src/middleware.ts`) that runs before every request. Used for Clerk authentication route protection.

---

## N

### Node (LangGraph)
A named function in a LangGraph graph. Each node is an agent that receives state and returns partial state updates.

---

## O

### Outline Generator
The 2nd agent in the pipeline. Generates a list of slide topics/outlines from the user's input topic.

### Ownership Check
The `getOwnedProject()` function that verifies a project belongs to the authenticated user before allowing any mutation.

---

## P

### Partialize
A Zustand `persist` middleware option that selects which state fields to persist. Used in `useSlideStore` to exclude `past` and `future` (undo/redo stacks) from localStorage.

### Presentation Generation Run
See [Generation Run](#generation-run).

### Project
The primary data model representing a presentation deck. Contains title, slide JSON, outlines, theme, publish status, and ownership.

### Project Initializer
The 1st agent in the pipeline. Creates the `Project` record in the database.

---

## R

### Route Group
A Next.js App Router convention using parenthesized directories `(groupName)` to organize routes without affecting the URL structure. Example: `(protected)`, `(pages)`, `(auth)`.

### RSC (React Server Component)
A React component that renders on the server, never ships JavaScript to the client. The default in Next.js App Router.

---

## S

### Server Action
A Next.js function marked with `"use server"` that runs on the server but can be called directly from client components. Used for all data mutations in Verto AI.

### Slide
A top-level presentation unit containing an `id`, `slideName`, `type` (layout), `slideOrder`, and a root `ContentItem` tree as `content`.

### SlideGenerationData
A per-slide tracking object populated incrementally by each agent during generation. Contains outline, content, layout type, image query, image URL, and compiled JSON.

### Soft Delete
A deletion strategy that sets `isDeleted = true` instead of removing the record. Projects use soft delete; hard delete is only available from the trash view.

### SSE (Server-Sent Events)
A web standard for pushing events from server to client over a single HTTP connection. Used for real-time generation progress streaming.

### StateGraph
The core LangGraph class for defining multi-agent workflows. Combines nodes (agents), edges (connections), and channels (state fields) into an executable graph.

### StreamingEventEmitter
A singleton class (`streamingEmitter`) that manages SSE event subscriptions per generation run. Stores event history for replay on reconnect.

---

## T

### Theme (Application)
The dark/light mode toggle managed by `next-themes`. Default: dark.

### Theme (Presentation)
A style configuration applied to slide content. Includes font family, colors, accent color, and background. Stored in `Project.themeName`.

### Turbopack
Next.js's Rust-based bundler replacing Webpack. Provides significantly faster development server starts and HMR.

---

## U

### Undo/Redo
The `useSlideStore`'s history system. Uses `past[]` and `future[]` stacks of `Slide[][]` snapshots. Every mutation pushes current state to `past` and clears `future`.

---

## W

### Webhook
An HTTP callback where an external service pushes events to your server. Used for Lemon Squeezy subscription events and Inngest function triggers.

### wrapNode
A higher-order function in `advanced-genai-graph.ts` that wraps each agent with progress tracking (database + SSE), error handling, and logging.

---

## Z

### Zod
A TypeScript-first schema validation library. Used extensively in the AI pipeline to validate LLM outputs against expected shapes (outlines, content, layout selections, image queries).

### Zustand
A minimal React state management library. Verto AI uses 6 Zustand stores for client-side state.
